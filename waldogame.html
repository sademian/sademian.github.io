Game Title:

Genre:

Survival Horror

Platform:

PC

Engine:

Unreal Engine 4

Theme:

Where's Waldo?

Game Inspiration:

Hello Neighbor

Premise:

You're trying to collect the hidden Waldo game pieces and make your way to the exit. The only issue is you're in an area full of Waldo look-alikes and the real Waldo is trying to kill you. Close encounters with Waldo require you to Sprint and hide in the environment to have Waldo lose your trail and get another opportunity to seek out the hidden game pieces. You can only make your way to the exit once you've collected all the Waldo game pieces.

Week of November 26, 2018

- In the following item spawning BP the "Volume" would refer to the area that is designated to spawn items. It will take a random X and Y and place the items at that desired location. This action will be designated on "SpawnNewItem" Event. This event will check that the X and Y coordinate do not hide inside assets in the environment. Each asset has a "NoSpawnItem" Box which sets the boundaries of each individual asset. If the X and Y coordinate match with the any point within the "NOSpawnBox" it will run the "SpawnNewItem" event again till it finds a location in the playing field.


- Items hiding with in objects resolution: In the following BP the a call on all five item event are done within 0.2 seconds.


- as previously the boundaries of the spawning box are picked at random X and Y coordinates and put into a vector.


- This vector will then be send to each individual items event handler which will then look if the coordinates that were produced are equal to any coordinates within the "NoSpawnBox" (Which are set on each of the assets within the game). If the coordinates match to any coordinate within the "NoSpawnBox" it will call on the event handler again to ask for another coordinate. if the coordinate does not match then the item will be placed in given location that was produced. This procedure is done throughout all 5 items.

Week of November 12, 2018

- We improved the crowd wandering AI. Before, the characters would pick a random location within a certain radius; their behavior felt unrealistic because sometimes they would make sharp turns. Now, the characters choose a random point on the radius of a circle in front of them; this limits the angle change. When the character is in a location where they can't move forward because they are blocked by an object or person, they then choose a random point on a circle behind them instead.
